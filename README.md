# Kubernetes Cluster Setup Using Kubeadm

This project documents the process of building a multi-node Kubernetes cluster from scratch using `kubeadm` on AWS EC2 instances.

The cluster uses an Ubuntu control-plane node, Red Hat Enterprise Linux worker nodes, CRI-O as the container runtime, and Calico as the CNI plugin.

## Step 1: Install Kubernetes and CRI-O

Run the installation script on the Ubuntu control-plane node:

```bash
chmod +x install-k8s-crio-Ubuntu.sh
./install-k8s-crio-Ubuntu.sh
```

The script installs and configures:

* CRI-O container runtime
* kubeadm
* kubelet
* kubectl
* Required kernel modules
* Required sysctl networking settings
* Swap disablement

---

## Step 2: Initialize the Kubernetes Control Plane

Run this command on the master node:

```bash
sudo kubeadm init \
  --control-plane-endpoint "<MASTER_PRIVATE_IP>:6443" \
  --apiserver-advertise-address "<MASTER_PRIVATE_IP>" \
  --cri-socket unix:///var/run/crio/crio.sock
```

Example:

```bash
sudo kubeadm init \
  --control-plane-endpoint "172.31.25.101:6443" \
  --apiserver-advertise-address "172.31.25.101" \
  --cri-socket unix:///var/run/crio/crio.sock
```

---

## Step 3: Configure kubectl Access

After initializing the cluster, configure `kubectl` for the current user:

```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown "$(id -u):$(id -g)" $HOME/.kube/config
```

Verify the cluster:

```bash
kubectl get nodes
```

---

## Step 4: Install Calico CNI

Install Calico networking:

```bash
kubectl apply -f https://raw.githubusercontent.com/projectcalico/calico/v3.32.0/manifests/calico.yaml
```

Check system pods:

```bash
kubectl get pods -n kube-system -o wide
```

---

## Step 5: Join Worker Nodes

Use the `kubeadm join` command generated by the master node.

Example:

```bash
sudo kubeadm join <MASTER_PRIVATE_IP>:6443 \
  --token <TOKEN> \
  --discovery-token-ca-cert-hash sha256:<HASH> \
  --cri-socket unix:///var/run/crio/crio.sock
```

Verify nodes:

```bash
kubectl get nodes -o wide
```

Expected result:

```text
NAME       STATUS   ROLES           VERSION
master     Ready    control-plane   v1.32.x
worker01   Ready    <none>          v1.32.x
worker02   Ready    <none>          v1.32.x
```

---

## Step 6: Deploy Sample HTTP Application

Create the ConfigMap from the custom HTML file:

```bash
kubectl create configmap booking-html \
  --from-file=index.html=booking-index.html
```

Deploy Apache HTTPD with NodePort service:

```bash
kubectl apply -f httpd-booking.yaml
```

Check pods and service:

```bash
kubectl get pods -o wide
kubectl get svc
```

Access the application in the browser:

```text
http://<EC2_PUBLIC_IP>:30080
```

## Troubleshooting Notes

### kubeconfig Permission Issue

If `kubectl` shows permission denied:

```bash
sudo chown -R ubuntu:ubuntu /home/ubuntu/.kube
chmod 700 /home/ubuntu/.kube
chmod 600 /home/ubuntu/.kube/config
```

---

### RHEL Worker Node resolv.conf Issue

If worker pods fail with:

```text
Failed to create pod sandbox: open /run/systemd/resolve/resolv.conf: no such file or directory
```

Update kubelet config on the RHEL worker nodes:

```bash
sudo sed -i 's#^resolvConf: .*#resolvConf: /etc/resolv.conf#' /var/lib/kubelet/config.yaml
sudo systemctl restart crio kubelet
```

---

### Calico BGP Issue

If Calico shows:

```text
BIRD is not ready: BGP not established
```

Allow node-to-node traffic in the AWS Security Group.

Recommended lab rule:

```text
Type: All traffic
Source: Same Security Group ID
```

Or allow these Calico ports/protocols:

```text
TCP 179
UDP 4789
Protocol 4
```

Then restart Calico:

```bash
kubectl -n kube-system delete pod -l k8s-app=calico-node
```
